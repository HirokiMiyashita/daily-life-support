# 減量プラン管理アプリ デザイン要件要約

## プロジェクト概要
- **アプリ名**: daily-life-support（減量プラン管理アプリ）
- **目的**: 半年で-20kgを達成するための毎日のやること・食事・ジムメニューを管理
- **プラットフォーム**: iOS（React Native + Expo）
- **ターゲットユーザー**: 26歳女性、在宅勤務（9:00-18:00）、朝ジムなし・夜ジム中心

## デザインコンセプト
- **シンプルで迷わず実行できるUI**
- **当日すぐ見れる**
- **記録して改善できる**
- **個人利用のため、シンプルで実用的**

## 画面構成（タブナビゲーション）

### 1. Today（今日）画面 - 最重要
**目的**: 今日のやることを一目で確認

**セクション構成**:
1. **ヘッダー**
   - 日付・曜日表示（例: 2026年1月4日 (土)）
   - 今日のタイプ表示（トレーニング日 / 有酸素日 / 休養日）

2. **今日のやること（チェックリスト）**
   - チェックボックス付きリスト
   - 毎日の共通項目（体重測定、食事、散歩、ストレッチなど）
   - トレ日追加項目（ジム、有酸素、トレ後プロテインなど）

3. **今日の食事メニュー**
   - 朝食（200-250kcal）
   - 昼食（500-550kcal）
   - 間食（100-150kcal）
   - トレ後（150-200kcal）- トレ日のみ
   - 夕食（400-450kcal）
   - 各食事に食材・量を表示

4. **今日のジムメニュー**（トレ日のみ）
   - 曜日別メニュー表示
   - 種目・回数・セット数
   - 有酸素設定（時間・強度）

5. **今日の目標サマリ**
   - カロリー: 1450-1550kcal（トレ日）/ 1300kcal（休み日）
   - タンパク質: 130g
   - 脂質: 40g以下
   - 歩数: 8,000歩

### 2. Week（週間）画面
**目的**: 週間プランを一覧表示

**構成**:
- 週の開始日・終了日表示
- 月〜日のカード一覧
  - 各日に日付・曜日
  - タイプ表示（トレーニング日/有酸素日/休養日）
  - 今日はハイライト表示
  - タップで詳細表示
- 週間サマリ
  - トレーニング日: X日
  - 有酸素日: X日
  - 休養日: X日

### 3. Logs（記録）画面
**目的**: 体重・ウエストの記録と履歴表示

**セクション構成**:
1. **体重ログ（毎朝）**
   - 入力フォーム（kg）
   - 現在の体重表示
   - 保存ボタン

2. **ウエスト（週1）**
   - 入力フォーム（cm）
   - 現在のウエスト表示
   - 保存ボタン

3. **7日平均**
   - 過去7日間の平均体重を大きく表示

4. **体重履歴（過去30日）**
   - 日付・体重のリスト表示
   - ウエスト記録がある場合は併記

### 4. Shopping List（買い物リスト）画面
**目的**: 今週必要な食材をカテゴリ別に表示

**構成**:
- 週の期間表示（月〜日）
- カテゴリ別食材リスト
  - **タンパク**: 鶏むね、鮭、卵など（数量表示）
  - **野菜**: 冷凍ブロッコリー、キャベツなど
  - **糖質**: 米、バナナなど
  - **その他**: 豆腐、納豆など
- 各食材に数量・単位を表示（例: 鶏むね 2.0kg、卵 20個）

## デザインスタイル

### カラースキーム
- **背景**: 白（#fff）
- **セクション背景**: 薄いグレー（#f5f5f5）
- **ボーダー**: ライトグレー（#e0e0e0）
- **テキスト**: ダークグレー（#333）、ミディアムグレー（#666）
- **タイプ別カラー**:
  - トレーニング日: 赤系（#ffebee背景、#c62828テキスト）
  - 有酸素日: 緑系（#e8f5e9背景、#2e7d32テキスト）
  - 休養日: 紫系（#f3e5f5背景、#7b1fa2テキスト）
  - 今日ハイライト: 青系（#e3f2fd背景、#2196F3ボーダー）

### タイポグラフィ
- **ヘッダー**: 24px、太字
- **セクションタイトル**: 20px、太字
- **本文**: 16px
- **サブテキスト**: 14-16px、グレー

### レイアウト
- **パディング**: セクション間 20px
- **ボーダー**: セクション間に1pxのライトグレーボーダー
- **カード**: 角丸12px、影なし、シンプルなボーダー
- **入力フォーム**: 角丸8px、ライトグレーボーダー

### UIコンポーネント
- **チェックボックス**: シンプルなチェックボックス（テキストベース）
- **ボタン**: 標準的なiOSスタイル
- **入力フィールド**: シンプルなテキスト入力、数値キーボード
- **カード**: 白背景、薄いボーダー、角丸

## ユーザーフロー

### 典型的な1日の流れ
1. **朝（07:30）**: Today画面を開く → 体重測定をチェック → 朝食確認
2. **昼（12:00）**: 昼食確認 → 食後散歩をチェック
3. **夕方（18:00）**: ジムメニュー確認 → トレーニング実行 → チェック
4. **夜（20:30）**: 夕食確認 → ストレッチチェック
5. **記録**: Logs画面で体重・ウエストを記録

### 週初めの流れ
1. **Week画面**: 週間プランを確認
2. **Shopping List画面**: 今週の買い物リストを確認・購入

## デザインの優先順位
1. **可読性**: 情報が一目で分かる
2. **操作性**: タップしやすい、入力しやすい
3. **シンプルさ**: 装飾を最小限に、実用性重視
4. **一貫性**: 全画面で統一されたデザイン言語

## 技術的制約
- React Native（Expo）で実装
- iOS向け（iPhone）
- タブナビゲーション（下部タブバー）
- スクロール可能なコンテンツ

## 参考情報
- 現在の実装は基本的な構造のみ（スタイリングは最小限）
- デザインの改善・洗練が必要
- ユーザーは26歳女性、シンプルで使いやすいデザインを好む
